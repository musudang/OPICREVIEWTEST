# OPIc(오픽) 시험 완전 분석 및 연습 웹앱 설계

> **작성일**: 2026-07-20
> **상태**: 리서치/제안 (구현 전 — 기술 스택·범위는 유저와 논의 후 확정)

---

## 1. 현재 상황과 목적

프로젝트는 SPARK+IPO 템플릿을 복제한 초기 상태로, `input/` · `proc/` · `src/` · `output/` 이 모두 비어 있다. 목표는 **OPIc 시험을 실전처럼 연습할 수 있는 웹사이트 프로그램**을 만드는 것이다.

시험 메커니즘(진행 순서, 문제 유형, 채점 기준)을 정확히 반영하지 않고 앱을 설계하면 다음과 같은 문제가 생긴다.

- 문제 순서·세트 구조가 실제 시험과 달라 "실전 감각"을 기르는 훈련 효과가 떨어짐
- 무제한 재녹음, 시간 무제한 등 실제 시험엔 없는 관대한 UX를 주면 실전에서 오히려 당황함
- 채점 기준(Function/Accuracy/Content·Context/Text Type)을 모르는 채로 피드백 기능을 만들면 유저에게 의미 없는 피드백을 주게 됨

이 문서는 (2) 시험 자체에 대한 상세 조사, (3) 연습 앱을 어떻게 만들지에 대한 설계 방안을 함께 정리한다.

## 2. OPIc 시험 상세 분석

### 2-1. 개요

| 항목 | 내용 |
|---|---|
| 주관 | ACTFL(미국외국어교육협회) 공인, 한국 시행사는 YBM(오픽 공식 홈페이지 opic.or.kr) |
| 성격 | Oral Proficiency Interview — computer. 정답이 있는 시험이 아니라 **실제 말하기 능력을 재는 Proficiency Test** |
| 응시 방식 | 컴퓨터 기반(CBT), 화면 속 가상 인터뷰어 '에이바(Ava)'와 1:1 대화 형식 |
| 결과물 | 응시자의 답변 음성을 ACTFL 공인 평가자(rater)가 청취 후 등급(NL~AH) 부여 |

### 2-2. 전체 진행 흐름

시험은 총 약 **60분**: **오리엔테이션 20분 + 본 시험 40분**.

```
[오리엔테이션 ~20분]
  1. 시험 안내 (화면/장비 테스트, 마이크 체크)
  2. Background Survey (배경 설문)
  3. Self Assessment (난이도 1~6 선택)
  4. 샘플 문제로 연습(오리엔테이션용, 채점 제외)

[본 시험 ~40분, 12~15문항]
  1번: 자기소개 (채점 제외)
  2~4번: 세트1 — 설문 기반 주제 콤보
  5~7번: 세트2 — 설문 기반 주제 콤보
     (중간에 난이도 재조정 기회 1회: "쉬운 문제 / 그대로 / 어려운 문제")
  8~10번: 세트3 — 돌발 주제 콤보
  11~13번: 세트4 — 롤플레이 (상황극)
  14~15번: 세트5 — 고난도 어드밴스 문제 (난이도 5~6 선택 시에만 출제)
```

문항 수는 난이도 선택에 따라 다르다.

| Self Assessment 난이도 | 출제 문항 수 |
|---|---|
| 1~2단계 | 12문항 |
| 3~6단계 | 15문항 |

### 2-3. Background Survey (배경 설문)

- 약 4개 대분류(현재 신분: 학생/직장인/무직 등, 거주 형태, 여가활동, 취미·관심사, 운동, 여행 경험 등) 아래 **약 20여 개 항목**을 자유롭게 선택
- **선택한 항목 = 설문 기반 질문의 출제 풀**이 되고, **선택하지 않은 주제 = 돌발 질문**으로 나올 수 있음
- 즉 설문 선택 자체가 "내가 어떤 주제로 시험 볼지"를 스스로 정하는 단계 → 연습 앱에서도 이 설문-문제 매핑 구조를 재현해야 실전과 같은 경험이 됨

### 2-4. Self Assessment (난이도 선택)

- 1~6단계 중 선택. 단계가 높을수록 더 어려운 문제(콤보셋 심화 질문, 롤플레이, 어드밴스 세트)가 출제됨
- 목표 등급별 권장 난이도 (커뮤니티 통설, 공식 보장 아님)
  - 3~4단계: IL~IM2 목표
  - 5~6단계: IH~AL 목표
- 본 시험 중간에 1회 난이도 재조정 기회가 주어짐

### 2-5. 문제 유형별 특징

| 유형 | 설명 | 세트 내 위치 |
|---|---|---|
| 자기소개 | "Tell me about yourself" — 채점 제외, 워밍업 성격 | 1번 |
| 콤보셋(3연타) | 한 주제에 대해 3문제 연속 출제. 통상 **① 묘사(description) → ② 습관/비교(habit or comparison) → ③ 과거 경험(past experience)** 순서. 세트 내 첫 문제 유형이 정해지면 나머지도 거의 고정된 패턴을 따름 | 2~10번 (세트1~3) |
| 롤플레이 | 상황이 주어지고 "질문하기", "문제 제기 후 대안 제시하기" 등 연기력·순발력 요구. 예: 호텔 예약 문제 해결, 친구에게 전화로 부탁하기. **스킵 시 감점 폭이 크다고 알려져 있어 반드시 시도해야 함** | 11~13번 |
| 돌발 질문 | 설문에서 고르지 않은 주제, 최근 사회 이슈·기술 변화 등 예측 불가 주제. 난이도가 올라갈수록 비중 증가 | 8~10번 및 전반에 산재 |
| 어드밴스 세트 | 난이도 5~6 선택자에게만 출제되는 고난도 문제(가정법, 사회적 이슈 의견 등) | 14~15번 |

### 2-6. 답변 진행 규칙 (UI/시간)

- 문제 음성은 **최대 2회까지 재청취** 가능
- 답변은 문제 청취 직후 **자동으로 녹음 시작** (재녹음/일시정지 불가 — 단발 녹음)
- 문항별 **제한 시간은 없음** — 응시자가 답변을 마치면 직접 다음으로 넘어감. 단, 전체 시험은 40분 내 완료해야 함
- 문제를 건너뛰고 싶으면 **약 10초 후 'Next' 버튼이 활성화**됨
- 통상 문제당 평균 1.5~2분을 목표로 시간 배분(자신 있는 주제·롤플레이에 더 투자하는 전략이 일반적)

### 2-7. 채점 방식

- 녹음된 답변 전체를 **ACTFL 공인 평가자(rater)**가 청취하여 홀리스틱(전체적) 방식으로 등급을 매김 — AI 자동채점이 아니라 **인간 평가자 채점**
- 평가 기준 4가지 (ACTFL Proficiency Guidelines 기반)

| 기준 | 내용 |
|---|---|
| **Function** (기능/과업 수행) | 질문이 요구하는 화행(과업)을 수행했는가 — 단순 서술/묘사/과거 서술/비교/설명/설득/가정 등 요구되는 기능 수준에 맞게 답했는가 |
| **Accuracy** (정확성) | 문법, 어휘, 발음, 유창성이 의사소통을 방해하지 않는 수준인가 |
| **Content/Context** (내용/맥락 적합성) | 질문의 의도와 상황 맥락에 맞는 내용으로 답했는가 |
| **Text Type** (담화 구성) | 단어 나열 수준인지, 문장 단위인지, 문단(paragraph) 단위로 논리적 흐름을 가진 발화인지 |

핵심 로직: **"일관되게 할 수 있는 최고 수준(강점의 패턴)"**과 **"그 이상에서 무너지는 지점(약점의 패턴)"**을 함께 보고 등급을 정함 → 한두 문제를 못해도 전체 패턴이 중요하고, 반대로 한두 문제를 잘해도 전체에서 유지가 안 되면 그 등급을 못 받음.

### 2-8. 등급 체계

ACTFL 10단계 척도 (낮은 순):

```
Novice        : NL(Novice Low) → NM(Novice Mid) → NH(Novice High)
Intermediate  : IL(Intermediate Low) → IM1/IM2/IM3(Intermediate Mid, 3단계 세분) → IH(Intermediate High)
Advanced      : AL(Advanced Low) → AM → AH(Advanced High)   ※ 오픽에서 국내 실무상 AL이 최상위 목표로 가장 흔히 언급됨
Superior      : S
```

| 등급 | 특징 (통설 기준) |
|---|---|
| NL~NH | 단어·구 수준 답변, 문장 구성 어려움 |
| IL | 일상 소재를 문장 단위로 말할 수 있는 수준. 국내 대기업 이공계 최소 지원 요건으로 흔히 언급됨 |
| IM1/IM2/IM3 | 국내 응시자 최다 분포 구간. 일상 소재는 자연스럽게, 복잡한 소재는 흔들림 |
| IH | 익숙하지 않은 복잡한 상황에서도 사건 설명·문제 해결이 대체로 가능하나 전 영역에서 일관되진 않음 (상위 약 7~30%) |
| AL | 모든 주요 시제로 서사·묘사 가능, 예상 못한 복잡한 상황에서도 문제 해결. 시제 일관성, 다양한 접속사·형용사 사용 (상위 약 3~7%) |

### 2-9. 고득점(AL/IH) 전략 요소 — 앱의 피드백 로직 설계에 참고

커뮤니티에서 반복적으로 언급되는 요소들이며, 연습 앱의 (규칙기반/AI) 피드백 항목 설계에 직접 활용 가능하다.

- 시제 일관성 (과거/현재/미래 전환의 정확성)
- 접속사 다양성 (because, although, however, moreover 등)
- 어휘 다양성/구체성 (good/nice 대신 impressive, exceptional 등)
- 롤플레이 논리 구조: 문제 인식 → 해결책 제안 → 재확인
- 돌발 질문 대처: 모른다고 솔직히 인정 후 아는 선에서 자연스럽게 전환하는 것도 유효한 전략
- 답변 분량(발화량)과 침묵/필러(uh, um) 비중

### 2-10. 유사 서비스 (참고용 벤치마크)

조사 중 확인된 기존 OPIc 연습 서비스: 오픽업(opicup.com), 오픽미(opicme.kr), INEAR, Ringle AI Prep, 스픽(Speak) 등. 공통 기능은 기출 유사 문제은행, 실전과 동일한 UI 재현(설문·난이도·재생 2회 등), 녹음 후 재생/스크립트화, AI 또는 자체 기준 피드백이다. 직접적 벤치마킹은 후속 논의에서 필요 시 진행.

## 3. 연습 웹앱 설계 — 방안 비교

### Option A: 정적 문제은행 + 셀프 녹음/재생 (채점 없음)

| 장점 | 단점 |
|---|---|
| 서버 불필요, 프론트엔드만으로 MVP 구현 가능 | 채점/피드백이 없어 "혼자 말하기 연습"에 그침 |
| 브라우저 MediaRecorder API로 즉시 구현 | 실력 향상 체감이 낮음 |
| 개발/운영 비용 거의 없음 | |

### Option B: 문제은행 + STT(음성인식) + 규칙기반 분석

| 장점 | 단점 |
|---|---|
| 답변을 텍스트로 변환해 발화량·속도·filler·시제/접속사 키워드 등 정량 피드백 가능 | 규칙기반이라 "내용이 맥락에 맞는가(Content/Context)"는 평가 못함 |
| STT는 브라우저 Web Speech API(무료) 또는 외부 API 선택 가능 | STT 정확도가 한국인 영어 발음에서 떨어질 수 있음 |
| LLM 없이도 동작 → 비용/키 관리 부담 적음 | |

### Option C: 문제은행 + STT + LLM 기반 정성 채점 ⭐ 최종 목표로 권장

| 장점 | 단점 |
|---|---|
| Function/Accuracy/Content·Context/Text Type 4대 기준별 코멘트 + 예상 등급 추정까지 가능 | LLM API 키·비용 필요, 서버(또는 서버리스 함수) 필요 |
| 실제 채점 로직과 가장 유사한 "패턴 기반" 피드백 제공 가능 | 실제 ACTFL 등급과 괴리 가능 → "참고용" 명시 필수 |
| 롤플레이 논리 구조, 돌발 대처 등 정성적 요소까지 평가 가능 | 개발 난이도·응답 지연 시간 가장 높음 |

## 4. 권장안: 단계적 접근 (A → B → C)

한 번에 Option C를 구현하기보다, **MVP(A) → 정량 피드백(B) → 정성 AI 피드백(C)** 순으로 단계적으로 확장하는 것을 권장한다. 이유: 시험 흐름 재현(오리엔테이션/설문/난이도/세트 구조/재생 2회/스킵 규칙)이 먼저 정확히 자리잡아야 그 위에 얹는 피드백 기능이 의미를 가지며, 채점 로직은 검증에 시간이 걸리는 영역이라 초반부터 정확도를 보장하기 어렵기 때문이다.

### 구현 범위 (레이어별)

- **문제은행(데이터)**: 주제(설문 카테고리) · 유형(콤보/롤플레이/돌발/어드밴스) · 난이도 · 세트 내 순서를 속성으로 갖는 JSON/DB 구조
- **프론트엔드**: 오리엔테이션 → 배경설문 선택 → 난이도 선택 → 세트별 문제 진행(재생 2회, 자동 녹음, 스킵 10초 룰, 중간 난이도 재조정) → 결과/기록 화면
- **오디오**: MediaRecorder API로 답변 녹음, 재생 UI
- **STT/피드백(Option B/C 단계)**: 음성 → 텍스트 변환 후 규칙기반 분석 또는 LLM 프롬프트로 4대 기준 피드백 생성
- **저장**: 로컬(IndexedDB, 계정 불필요) 또는 서버(계정별 이력 관리) — 규모에 따라 선택

## 5. 적용 대상 (기능 범위)

| 기능 | 우선순위 |
|---|---|
| 배경 설문(Background Survey) 선택 UI | 필수 |
| 난이도(Self Assessment) 선택 | 필수 |
| 세트 구조 기반 문제 출제(콤보/롤플레이/돌발/어드밴스) | 필수 |
| 문제 음성 재생(2회 제한) — TTS 또는 사전 녹음 필요 | 필수 |
| 답변 녹음 + 자동 진행, 스킵 규칙 | 필수 |
| 시험 전체 타이머(40분) | 권장 |
| 녹음 재생/다시듣기(연습 후) | 권장 |
| STT 스크립트화 + 정량 피드백(발화량, 시제/접속사 키워드) | 확장 |
| LLM 기반 4대 기준 정성 피드백 + 예상 등급 | 확장 |
| 학습 이력/오답노트/취약 유형 통계 | 확장 |

---

## 6. 구현 단계 (Phase)

1. **Phase 1**: 문제은행 데이터 구조 설계 + 정적 연습 모드(주제별 골라서 자유 연습, 시간제한 없음)
2. **Phase 2**: 실전 시뮬레이션 모드 — 오리엔테이션·설문·난이도 선택 → 세트 순서대로 12/15문항 진행, 재생 2회·스킵 10초 규칙 재현
3. **Phase 3**: 답변 녹음/재생, 회차별 기록 저장
4. **Phase 4**: STT 연동 + 정량 피드백(말속도, 침묵/필러 비율, 시제·접속사 다양성)
5. **Phase 5**: LLM 기반 4대 기준(Function/Accuracy/Content·Context/Text Type) 코멘트 + 참고용 예상 등급

---

## 7. 리스크 및 고려사항

- **저작권/상표**: 실제 오픽 문항이나 음성을 그대로 복제하면 ACTFL/YBM 저작권 및 "OPIc" 상표 문제 소지가 있음 → 문제은행은 **실제 기출을 참고한 자체 제작 유사 문제**로 구성하고, 화면 문구도 "OPIc 공식 시험이 아닌 연습 프로그램"임을 명시해야 함
- **마이크/녹음 권한**: 브라우저별 MediaRecorder 지원 포맷 차이(webm/mp4 등), HTTPS 필수(마이크 권한은 secure context에서만 동작)
- **STT 정확도**: 한국인 영어 발음 인식률 이슈 → 브라우저 내장 Web Speech API 무료지만 정확도 낮을 수 있음, 외부 STT API는 비용 발생
- **LLM 채점 신뢰도**: 실제 ACTFL 등급과 다를 수 있으므로 "참고용 예상 등급"임을 UI에 명확히 고지 필요
- **개인정보**: 음성 녹음을 서버에 저장한다면 처리방침·보관기간·삭제 정책 필요

## 8. 다음 논의 사항 (유저와 확정 필요)

CLAUDE.md 규칙상 기술 스택·프로젝트 유형은 임의로 정하지 않고 유저 확인이 필요하다. 다음 항목을 대화로 정한다.

1. **기술 스택**: 웹앱으로 확정됨(React+Vite? Next.js? 다른 프레임워크?), 백엔드 필요 여부
2. **채점 방식 범위**: MVP를 Option A(녹음만)로 시작할지, 처음부터 STT/LLM 피드백(Option B/C)까지 포함할지
3. **LLM 사용 여부**: 채점에 AI API(Claude 등)를 쓸지 — 쓴다면 API 키 준비 필요
4. **계정/로그인 필요 여부**: 기록을 로컬에만 둘지, 서버에 유저별로 저장할지
5. **문제은행 소스**: 직접 자체 제작할지, 유저가 가진 기출/참고 자료를 `input/`에 넣어 활용할지
