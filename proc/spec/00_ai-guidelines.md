# AI 명령지침 (Core Rules)

> **작성일**: 2026-07-20
> **적용 대상**: 이 프로젝트(OPIc 연습 웹앱)를 개발하는 모든 AI 에이전트

## 필수 준수 사항

1. **저작권 준수**: 실제 OPIc 시험 문항·음성·화면 UI를 그대로 복제하지 않는다. 문제은행은 `proc/research/20260720_오픽시험분석및연습웹앱설계.md`의 조사 내용을 참고해 **자체 제작한 유사 문제**로 구성한다.
2. **공식 시험 아님 고지**: 모든 화면(특히 결과/등급 화면)에 "OPIc 공식 시험이 아닌 연습용 프로그램이며, 실제 채점 결과와 다를 수 있다"는 문구를 노출한다.
3. **LLM 채점은 참고용**: LLM 기반 등급·피드백은 반드시 "참고용 예상 등급(AI 추정)"으로 표기하고, 실제 ACTFL 공인 평가와 동일하지 않음을 명시한다.
4. **API 키 보호**: LLM/STT API 키는 Express 백엔드에서만 사용한다. 프론트엔드(브라우저) 번들에 API 키를 절대 포함하지 않는다.
5. **로컬 우선 저장**: 학습 기록(녹음, 이력, 피드백)은 브라우저 로컬(IndexedDB)에만 저장한다. 백엔드는 상태를 갖지 않는 API 프록시 역할만 한다 — 백엔드에 유저 데이터를 영속 저장하지 않는다.
6. **단계적 구현**: `proc/spec/08_roadmap.md`의 Phase 순서를 따른다. 뒷 단계 기능(STT, LLM 채점)을 앞당겨 구현하며 과설계하지 않는다.
7. **명세 우선**: 요구사항이 바뀌면 `/update-spec`으로 `proc/spec/`을 먼저 갱신한 뒤 개발한다. 복잡한 작업은 `/update-plan`으로 `proc/plan/`에 계획을 남기고 진행 상황을 갱신한다.
8. **개인정보 최소화**: 로그인·회원가입 없이 동작하는 것을 기본으로 한다(로드맵 참고). 녹음 데이터를 서버로 전송할 때도 채점 처리 후 저장하지 않고 폐기한다.

## 기술 판단 기준

- 기술 스택·인프라 선택지가 명세에 없거나 모호하면 임의로 정하지 않고 유저에게 확인한다.
- 새 외부 의존성(라이브러리, API 서비스) 도입 전 비용·유지보수 부담을 유저에게 공유한다.
