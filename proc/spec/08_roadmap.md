# 단계별 개발 로드맵

> **작성일**: 2026-07-20

## Phase 1 — 문제은행 + 자유 연습 모드 (MVP) — 완료

- 범위: 문제은행 JSON 시드 데이터(카테고리별 콤보 세트, 롤플레이, 돌발, 어드밴스 최소치), 자유 연습 화면(이름 입력, 문항수 5/10 선택, 주제/유형 필터), 녹음(MediaRecorder) + 재생, 세션 종료 직후 규칙기반 즉시 점수 화면
- 검증 기준
  - [x] Vite+React+TS 프로젝트 초기화 완료 (`app/`), Express 백엔드 스캐폴드 (`svr/`)
  - [x] 문제은행 최소 시드(콤보 6세트, 롤플레이 4개, 돌발 6개, 어드밴스 3개, 자기소개 1개, 총 32문항) 등록
  - [x] 2차 확장(2026-07-20): 실제 Background Survey Part1~4 taxonomy 전 영역 커버 — 콤보 32세트(96문항), 롤플레이 6개, 돌발 18개(기본 6 + 고난도 시사·의견 12), 어드밴스 5개, 자기소개 1개 = 총 126문항. 고난도 돌발은 난이도 5~6 전용
  - [x] 이름 입력 → 문항수 선택 → 필터 → 랜덤 셔플 출제 → 재생(TTS) → 녹음 → 저장 → 다음 전체 흐름 동작 확인
  - [x] 녹음 데이터 IndexedDB 저장/조회 확인 (홈 화면 최근 기록에 이름 표시)
  - [x] 세션 종료 직후 규칙기반(완료율·평균 답변 길이 기반) 즉시 점수 화면 동작 확인
  - [ ] 실제 브라우저(마이크 권한 허용)에서 녹음·재생 실사용 테스트 — 개발 프리뷰 샌드박스는 마이크 접근이 차단되어 미검증

## Phase 2 — 실전 시뮬레이션 모드

- 범위: 배경설문 → 난이도선택 → 세트 순서(1,2세트 설문/3세트 돌발/4세트 롤플레이/5세트 어드밴스) 진행, 재생 2회 제한, 스킵 10초 룰, 중간 난이도 재조정, 전체 타이머
- 검증 기준
  - [ ] 난이도별 문항수(12/15) 분기 동작
  - [ ] 설문 선택 항목이 세트1~2 출제 풀에 반영됨
  - [ ] 재생 횟수 제한/스킵 딜레이 UI 동작

## Phase 3 — 기록 관리

- 범위: 회차별 결과 저장/조회, 기록보관함 화면, 회차 상세(문항별 재생)
- 검증 기준
  - [ ] `/history` 리스트, `/history/:id` 상세 화면 동작
  - [ ] 로컬 데이터 스키마(`attempts`/`responses`) 기준 정합성 확인

## Phase 4 — STT 정량 피드백

- 범위: STT 연동(방식은 `07_tech-stack.md` 결정에 따름), 정량 지표(발화량/WPM/호흡당 문장길이/필러비율/시제·접속사 다양성) 계산, 목표등급 벤치마크 기준표(자체 추정치) 정의
- 검증 기준
  - [ ] 녹음 → 텍스트 변환 성공률 확인(샘플 테스트)
  - [ ] 정량 지표(WPM/호흡당 문장길이/발화량 포함) 계산 로직 단위 테스트
  - [ ] 목표등급별 벤치마크 기준표(`benchmarks`) 데이터 확정 — 외부 서비스 수치 복제 금지, 자체 추정 명시

## Phase 5 — LLM 정성 피드백 + 결과 화면 고도화

- 범위: Express 백엔드에 LLM 프록시 엔드포인트 구현. (1) ACTFL 4대 기준(Function/Accuracy/Content·Context/Text Type) + 보조 4축 레벨바(Complexity/Accuracy/Fluency/Coherence, 발음 제외) 판정, (2) 주제 적합도 경고, (3) 문법 오류 원문→교정문 대비(오디오 타임스탬프 포함), (4) 롤플레이 논리 구조 평가, (5) "목표등급 받기 위한 Checklist" 결과 화면(Worst/Best/목표평균 비교 차트 포함) 구현, (6) 다회차 집계 기반 "가장 못한 유형 및 주제" 카드
- 검증 기준
  - [ ] API 키가 프론트엔드에 노출되지 않음 확인
  - [ ] 롤플레이 문제에 대해 논리 구조(문제인식→해결책→재확인) 평가 포함 확인
  - [ ] "AI 추정, 참고용" 고지 문구 모든 피드백 화면에 노출
  - [ ] Checklist 화면의 비교 차트가 실제 벤치마크 데이터로 렌더링됨
  - [ ] 2회차 이상 데이터 축적 시 "가장 못한 유형 및 주제" 집계 정상 동작

## Phase 6 — (확장, 미착수) 발음 분석

- 범위: 텍스트 기반이 아닌 별도 음성 분석(STT 모델의 음소 단위 confidence 등)을 활용한 Pronunciation 축 추가. 착수 여부는 후속 논의.
