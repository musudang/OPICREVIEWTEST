# 디자인 시스템 (초안)

> **작성일**: 2026-07-20
> **상태**: 초안 — 실제 UI 작업 시작 전 유저와 톤 확정 필요

## 컨셉

시험 시뮬레이션에 집중하는 실용적·차분한 톤. 화려한 마케팅 UI보다 "실제 시험 화면과 유사한 신뢰감"을 우선한다.

## Color Palette (제안, 확정 아님)

| 용도 | 색상(예시) |
|---|---|
| Primary(진행/녹음 상태) | 차분한 블루 계열 |
| Recording 상태 강조 | 레드 계열(녹음 중임을 명확히) |
| 배경 | 뉴트럴 그레이/화이트, 다크모드 지원 여부는 논의 필요 |
| 경고/고지(공식 시험 아님 배너) | 앰버 계열 |

## Typography

- 제목: 굵고 명확한 산세리프
- 본문/문제 텍스트: 가독성 높은 폭 넓은 산세리프, 영어 문제 텍스트는 별도 줄바꿈 규칙 고려
- 강조(타이머, 진행 인디케이터): 숫자 가독성 높은 폰트

## Layout

- 문제 진행 화면은 좌우 여백 최소화, 중앙 단일 컬럼 집중형 레이아웃(시험 중 산만함 방지)
- 결과 화면은 탭/아코디언으로 4대 기준 피드백을 분리 노출

> 색상 코드·폰트 패밀리 등 구체값은 실제 프론트엔드 구현 착수 시 확정한다.
